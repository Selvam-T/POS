# Development Tools

These scripts support development, diagnostics, manual checks, and database maintenance. They are not imported or required by the running POS application. Run commands from the `Project` directory unless noted otherwise.

## Database

- `database/clear_cash_outflows_table.py`: Deletes all cash outflow records after an explicit `YES` confirmation and resets their sequence.
- `database/clear_receipt_tables.py`: Deletes all receipt, receipt item, and receipt payment records after an explicit `YES` confirmation.
- `database/clear_users_table.py`: Deletes all users after an explicit `YES` confirmation and resets their sequence.
- `database/query_all_tables_in_db.py`: Lists database tables, sample rows, columns, and row counts.
- `database/query_cash_outflow_tables.py`: Displays recent cash outflows and the total row count.
- `database/query_product_table.py`: Compares selected vegetable product records with application product lookup results.
- `database/query_receipt_tables.py`: Displays up to ten rows from each receipt-related table.
- `database/query_users_tables.py`: Displays recent users and the total user count.

The three `clear_*.py` scripts permanently remove data from the active database. Back up the database before using them.

## Diagnostics

- `diagnostics/list_fonts.py`: Writes the system fonts visible to Qt into `diagnostics/fonts_available.log`.
- `diagnostics/fonts_available.log`: Machine-specific output generated by `list_fonts.py`; it is excluded from Git.
- `diagnostics/font_preview.py`: Opens an interactive Qt window showing installed font families.
- `diagnostics/list_func_in_file.py`: Prints function definitions and line numbers from the configured source file.

## Manual Tests

- `manual_tests/scanner_check.py`: Opens a small Qt application that verifies barcode scanner signal handling.
- `manual_tests/preview_paynow_qr.py`: Generates and displays a PayNow QR preview image for visual/manual checking.

These are interactive checks, not automated pytest tests. Automated tests remain in `tests/`.

## Maintenance

- `maintenance/format_assets.py`: Formats Qt `.ui` files and QSS stylesheets; requires `lxml` and `jsbeautifier`.
- `maintenance/normalize_products.py`: Reports inconsistent product casing in dry-run mode and updates the database only when `--apply` is supplied.
- `maintenance/write_trial_build_timestamp.py`: Refreshes `modules/runtime/trial_build.py` with the current UTC timestamp before creating a trial executable.

Examples:

```cmd
python dev_tools\database\query_all_tables_in_db.py
python dev_tools\manual_tests\scanner_check.py
python dev_tools\manual_tests\preview_paynow_qr.py
python dev_tools\maintenance\format_assets.py
python dev_tools\maintenance\normalize_products.py
python dev_tools\maintenance\write_trial_build_timestamp.py
```
